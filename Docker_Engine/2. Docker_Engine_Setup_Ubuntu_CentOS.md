Ubuntu Installation
~~~~~~~~~~~~~~~~~~~~~~~~~

1. Update packages
> sudo apt update
(bcoz ubuntu package metadata may be outdated)

2. Install Required Packages
> sudo apt install ca-certificates curl
(curl -> download files
ca-certificates -> SSL verification)

3. Add Docker GPG Key
> sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
(verifies packages are genuinely from Docker)
without GPG verification, security risk

> sudo chmod a+r /etc/apt/keyrings/docker.asc

4. Add Docker repository
> echo \
"deb [arch=$(dpkg --print-architecture) \
signed-by=/etc/apt/keyrings/docker.gpg] \
https://download.docker.com/linux/ubuntu \
$(lsb_release -cs) stable" \
| sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

(Ubuntu's repository may contain old Docker version, Docker official repo contains Latest stable docker)

5. Update again
> sudo apt update
(Now Ubuntu recognizes Docker repository)

6. Install Docker
> sudo apt install -y \
docker-ce \
docker-ce-cli \
containerd.io \
docker-buildx-plugin \
docker-compose-plugin

(docker-ce    --->   Docker Comunity Edition Engin contains dockerd
docker-ce-cli  --->   contains docker command
containerd.io   ---->  installs containerd
buildx  --> advanced docker builds
compose plugin  -->  instals docker compose)

7. Verify Installation
Check service:
> sudo systemctl status docker

Check version:
> docker version

Check detailed information:
> docker info

Run first container:
docker run hello-world

Scenarios
~~~~~~~~~~~~~

(Cannot conect to Docker Daemon)
> systemctl status docker
> sudo systemctl start docker

(docker ps
permission denied while trying to connect to Docker daemon socket)
> groups
User not in docker group
> sudo usermod -aG docker $USER

Docker Directories

Docker Service
~~~~~~~~~~~~~~~~~
systemd service:
docker.service
> systemctl status docker

ex:
docker.service
      ⬇
starts
      ⬇
dockerd process


Docker Daemon
~~~~~~~~~~~~~~~~~~
Actual running process:
dockerd
> ps -ef | grep dockerd


Architecture:
docker.service
           ⬇
dockerd
           ⬇
containerd
           ⬇
runc


Docker Service is the systemd unit that starts and manages Docker Daemon.
Docker Daemon (dockerd) is the actual process responsible for managing containers, images, networks, and volumes.

Docker Daemon configuration stored at:
/etc/docker/daemon.json

docker.service -> dockerd -> Read daemon.json  -> Apply configuration  -> Start Engine

Viewing Current Docker Configuration
> docker info

Docker Startup Configuration

check startup status
> systemctl is-enabled docker

Enable docker at boot
> sudo systemctl enable docker
(now docker start automatically at reboot)

Disable Auto Start
> sudo systemctl disable docker

...................................................................................................................................................................
Restart Docker After Configuration Changes
Whenever daemon.json changes:
> sudo systemctl restart docker

Docker doesnot automatically reload daemon.json 
you must restart Docker

Docker DNS Configuration
~~~~~~~~~~~~~~~~~~~~~~~~~~

production issue: containers cannot resolve domains
> ping google.com fails inside containers

Configure DNS:
Edit
> sudo vi /etc/docker/daemon.json
ex:
{
  "dns": ["8.8.8.8", "1.1.1.1"]
}

> sudo systemctl restart docker

Verify:
> docker run -it ubuntu bash
> cat /etc/resolv.conf
nameserver 8.8.8.8
nameserver 1.1.1.1

Configuring Insecure Registries
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Normally Docker uses HTTPS
ex:  https://registry.company.com

Internal company registry:
http://10.10.10.5:5000
No TLS
Docker refuses connection

Configure
{
  "insecure-registries": [
     "10.10.10.5:5000"
  [
}

> sudo systemctl restart docker

verify:
> docker info

Configuring Docker Logging
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

By default:
> json-file

check: 
> docker info
Logging Driver: json-file


Configure Log Rotation:
daemon-json:
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "5"
  }
}

means 100MB per log file , keep 5 files

Docker Debug Mode
~~~~~~~~~~~~~~~~~~~
Useful for troubleshooting

Enable:
{
    "debug": true
}

> systemctl restart docker

Check logs:
> journalctl -u docker

Disable afterwards
{
  "debug": false
}
bcoz debug logs consume disk space

Docker Data Directory
~~~~~~~~~~~~~~~~~~~~~~~
> /var/lib/docker
contains images, containers, networks, volumes, layers

check size:
> du -sh /var/lib/docker


Change Docker Root Directory:
{
  "data-root": "/docker-data"
}
> systemctl restart docker
> docker info

Cgroup Driver Configuration
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Imp for k8s
ex:
{
  "exec-opts": ["native.cgroupdriver=systemd"]
}

verify:
> docker info | grep Cgroup

k8s expects docker, kubelet to use same cgroup driver , mismatch causes node issues

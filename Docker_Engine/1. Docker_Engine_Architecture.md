Components of Docker Engine
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
1. Docker CLI
CLI - command line interface
> docker run nginx
CLI is a client, it does not create containers
CLI only sends a request to Docker Daemon using Docker Rest API

2. Docker Daemon (dockerd)
brain of docker
service name - dockerd

> ps eef | grep dockerd
or
> sudo systemctl status docker

responsibilities: pull images, create containers, manage volumes, manage networks, start containers, stop containers, 

without dockerd, docker commands won't work.


3. Docker REST API
CLI and Daemon communicate through REST API

Socket:
/var/run/docker.sock
> ls -l /var/run/docker.sock

4. containerd
Docker internally uses containerd for container lifecycle management

responsibilities: containerd handles pull images, store images, start containers, stop containers, manage runtime

> ps -ef | grep containerd

Earlier, docker became hube and monolithic
to simplify architecture, docker extracted container management into containerd

5. runc
lowest-level container runtime
responsible for actually creating container

when container creates, runc creates namespaces, cgroups, process isolation using linux kernel features

> runc --version

Linux Kernel Features Used by Docker
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Docker uses linux kernel features mainly:
1. Namespaces
2. Cgroups
3. Union Filesystem


Namespaces
................
Provides isolation
Container sees own process list, network, filesystem, hostname even though sharng same kernel


Types of Namespaces
1. PID Namespace
provides process isolation

> ps -ef (in container)
shows only container processes not host processes

2. Network Namespace
provides separate networking
container gets own IP, own Routing Table, own Interfaces

3. Mount Namespace
provides seaprate filesystem view
each container sees its own filesystem

4. Hostname Namespace
Allows
> hostname
inside container to be different from host

5. User Namspace
provides user isolation
maps container users to host users
secuirty enhancement


Cgroups (Control Groups)
..........................
Cgroups limit resources
ex: CPU, RAM, Disk I/O, Network

Union Filesystem (OverlayFS)
..................................................
Images consist of layers
Each instruction creates a layer

Image Layer Structure
Layer 4   App Files
Layer 3   Nginx
Layer 2   apt update
Layer 1   Ubuntu Base
Docker combines layers using: OverlayFS
